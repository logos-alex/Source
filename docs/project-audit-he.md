# סקירה טכנית מקיפה — פרויקט Source

## מה הפרויקט הזה?
זהו אתר סטטי מבוסס Eleventy שמרכז כתבים אפוקריפיים בעברית (עם שיוך לשפת מקור ולדמות מקראית), ומייצר ניווט, חיפוש, ועמודי תוכן ללא שרת אפליקטיבי בזמן ריצה.

## סטאק וטכנולוגיה
- **Static Site Generator:** Eleventy 3 (`@11ty/eleventy`).
- **תבניות:** Nunjucks (`.njk`) עם Layout בסיסי ורכיבי include.
- **חיפוש סטטי:** Pagefind (אינדוקס לאחר build).
- **צד לקוח:** JavaScript וניהול מצבים ב־`localStorage` (ערכת נושא, מצב קריאה).
- **נתוני קטלוג:** JSON תחת `src/_data`.

## מבנה המערכת בפועל
- `src/_data/` — קונפיגורציה גלובלית וקטלוגים (כתובת אתר, pathPrefix, דמויות, ספרים).
- `src/_includes/` — קומפוננטות תצוגה מרכזיות (base, breadcrumbs, text-page, book-index).
- `src/texts/<language>/<book>/` — תוכן הספרים והפרקים בקבצי Markdown.
- `scripts/` — כלי QA/אימות מבנה, frontmatter, קישורים, קטלוגים ודוחות תוכן.
- `docs/` — דוחות נלווים (למשל סטטוס תוכן).

## איך האתר עובד מקצה לקצה
1. Markdown + frontmatter נטענים ע"י Eleventy.
2. תבניות Nunjucks מרנדרות עמודים תחת `_site`.
3. אחרי build רץ Pagefind ומייצר אינדקס חיפוש סטטי.
4. בזמן ריצה בדפדפן, `site.js` מפעיל:
   - dark mode,
   - reading mode,
   - תפריט ניווט מובייל,
   - התנהגות dropdown ל־TOC.

## נקודות חוזק
- ארגון תוכן ברור לפי שפה/ספר/פרקים.
- שכבת בדיקות סקריפטים רחבה יחסית (ולא רק build בסיסי).
- תמיכה מלאה ב־RTL וב־pathPrefix לפריסה תחת GitHub Pages.
- חיפוש Pagefind קיים ומחובר לעמוד ייעודי.

## צווארי בקבוק וסיכונים שכדאי לטפל
1. **תלות בנתיב בינארי קשיח ב־CI**
   - קודם `ci-verify` הריץ ישירות `node_modules/.bin/...`, מה שעלול להישבר בסביבות שונות.
   - עודכן להרצה דרך `npm exec -- ...` כדי לשפר ניידות ויציבות.
2. **אי-אחידות סביב התקנת תלויות**
   - קיימים `package-lock.json` וגם `pnpm-lock.yaml`.
   - מומלץ לקבע מנהל חבילות אחד ל־CI (npm או pnpm) ולהסיר חוסר עקביות.
3. **רעש סביב משתנה סביבה npm_config_http-proxy**
   - מופיעה אזהרה בכל ריצה.
   - כדאי לנקות מהסביבה או מה־CI vars כדי למנוע תקלות עתידיות ב־npm major הבא.
4. **נוכחות `node_modules` בתוך הריפו**
   - מקשה תחזוקה ועלול לגרום לשבירות לינקים/סימלינקים.
   - מומלץ לא לכלול `node_modules` ב־git ולהסתמך על lockfile + install ב־CI.

## המלצות עבודה להמשך (אופטימלי)
- להגדיר Pipeline יחיד וברור:
  - `npm ci` → `npm run ci:verify` → deploy.
- להמשיך ב־content governance:
  - חובת `frontmatter` אחיד לכל פרק,
  - בדיקת consistency מול `sources-catalog.json` בכל PR.
- לשפר observability:
  - לשמור artifacts של `_site` ו־Pagefind index בבדיקות CI לצורך דיבוג.
- לשלב בדיקת קישורים תקופתית גם על האתר הפרוס (לא רק `_site` מקומי).

## סיכום מנהלים
הבסיס הטכני של הפרויקט טוב מאוד לפרויקט ארכיון טקסטואלי סטטי: ארכיטקטורה מסודרת, תמיכה חזקה בעברית/RTL, ואוטומציות אימות טובות. ההשקעה העיקרית מכאן צריכה להיות על הקשחת תהליך ה־CI/Dependencies ושמירה על עקביות תוכן.
