# מפת עומק לפרויקט Source (Eleventy)

## 1) מהות הפרויקט
האתר הוא ארכיון דיגיטלי עברי לכתבים אפוקריפיים/פסאודו-אפיגרפיים, עם דגש על הנגשה לקהל קורא עברי: טקסטים, הקדמות, ולעיתים ביאורים/הערות. השדרה הרעיונית משלבת:
- שימור והנגשה של טקסטים שנדחקו מהקאנון.
- תרגום/עיבוד לשפה עברית וקריאה מחקרית.
- ניווט כפול: לפי שפת מקור ולפי דמות מקראית.

## 2) ארכיטקטורה טכנית

### סטאק
- Eleventy 3.1.2 כ-SSG.
- Nunjucks לתבניות.
- Pagefind לחיפוש סטטי.
- CSS/JS ונילה ללא framework צד-לקוח כבד.

### פקודות עבודה
- `npm run dev` מפעיל שרת פיתוח על פורט 5000 עם `pathPrefix=/Source/`.
- `npm run build` מריץ Eleventy ואז אינדוקס Pagefind לתיקיית `_site/pagefind`.
- פריסה מיועדת ל-GitHub Pages/Netlify עם אותו `pathPrefix`.

### נקודת בקרה קריטית
כל הניווט מבוסס `pathPrefix=/Source/` ומטופל גם בקונפיג (`.eleventy.js`) וגם במטא־דאטה (`src/_data/site.json`). זה חיוני כדי למנוע קישורים שבורים בפריסה תחת תת-נתיב.

## 4) שכבת תבניות וניווט

### layout בסיס
`base.njk` כולל:
- כותרת אתר, תפריט ניווט עליון קבוע, dark mode, reading mode.
- breadcrumbs רכיביים.
- footer עם קישורי sitemap/robots.
- טעינת Google Translate.

### דפי קטגוריה
- `texts.njk`: שער "לפי מקור".
- `by-figure.njk`: יצירה דינמית בפגינציה לכל דמות.
- `category-page.njk`: תבנית למבואי ספרים עם תוכן עניינים אוטומטי.
- `text-page.njk`: תבנית פרק עם progress bar, הערות, וניווט פרק קודם/הבא.

### לוגיקת אוספים
ב-`.eleventy.js` קיימים שני פילטרים מרכזיים:
- `bookPages`: מאתר את כל פרקי הספר הרלוונטיים, מסנן drafts, וממיין לפי `pageNumber`.
- `findIndexByUrl`: מאפשר לחשב מיקום הפרק הנוכחי בתוך הרצף.

## 5) חיפוש ו-SEO

### חיפוש
- `search.njk` מפעיל PagefindUI עם fallback לנתיבי סקריפט שונים.
- מוגדר `data-pagefind-body` בתבניות תוכן, כך שהאינדוקס מתמקד בגוף הטקסט.
- בבדיקה האחרונה: 82 עמודים מאונדקסים, ~8,886 מילים בעברית.

### SEO
- `sitemap.njk`, `robots.txt`, מטא תגיות וקנוניקל ב-layout.
- האתר RTL מלא עם `lang="he"` ויישור מותאם לעברית.

## 6) מצב תוכן בפועל (מבט מהיר)
- קיימות כמה משפחות טקסטים בשפות שונות (סלאבית, ארמית, עברית, ערבית, געז).
- "חזון אברהם" כולל לפחות שני נוסחים (A/B) עם חלוקה לפרקים רבים.
- "חזון דניאל הסורי-ארמי" מכיל עשרות פרקי תוכן.
- דפי קטגוריה לפי דמות/מקור כבר פעילים ויוצרים מסלול למידה ברור.

## 10) מסקנה
הפרויקט בשל ומבוסס היטב כ"ארכיון מחקרי-ציבורי" סטטי: טכנית הוא יציב, מהיר, וניתן להרחבה; תוכנית הוא כבר מחזיק ליבה משמעותית של חיבורים עם מסלולי ניווט הגיוניים. הצעד המשמעותי הבא הוא סטנדרטיזציה של מודל התוכן והעמקת שכבת הביאור, כדי להפוך מאוסף איכותי לתשתית מחקרית סדורה לאורך זמן.
