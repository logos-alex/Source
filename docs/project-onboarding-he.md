# מעבר ראשוני על הפרויקט (Onboarding)

## מה הפרויקט עושה בפועל
זהו אתר סטטי (Eleventy) שמציג קורפוס טקסטים אפוקריפיים בעברית, עם ניווט לפי מקור (שפה) ולפי דמות מקראית.
התוכן נשמר בפורמט Markdown ומעובד בזמן build לעמודים סטטיים תחת `_site`.

## איך המערכת בנויה
- **מנוע בנייה:** Eleventy 3.
- **תבניות:** Nunjucks (`src/_includes`).
- **מודל תוכן:** `src/texts/<source>/<book>/index.md` + פרקי `page-*.md`.
- **נתוני קטלוג:** `src/_data/sources-catalog.json`, `src/_data/languages.json`, `src/_data/figures.json`.
- **חיפוש:** Pagefind (`src/search.njk` + אינדוקס אחרי build).
- **בדיקות איכות:** אוסף סקריפטים תחת `scripts/` להרצת ולידציות על frontmatter, קישורים, פרמלאינקים ותוכן לשחרור.

## איך עובדים מקומית
- `npm run dev` — פיתוח מקומי עם `pathPrefix=/Source/`.
- `npm run build` — בניית האתר + אינדוקס חיפוש.
- `npm run ci:verify` — שרשרת הוולידציות המלאה (כמו CI).
- `npm run report:content` — מפיק דוח סטטוס תוכן ל-`docs/content-status-he.md`.

## מה אומת בסקירה הזו
הרצתי את שרשרת האימות המלאה (`npm run ci:verify`) והיא עברה בהצלחה:
- frontmatter
- מבנה טקסטים
- עקביות קטלוג
- כותרות פרקים
- permalinks
- placeholders
- build + pagefind
- release content
- path prefix
- built links

## נקודות שדורשות תשומת לב
1. **אזהרת npm סביב `http-proxy`** — בוצעה הקשחה ב־CI כדי לנטרל את המשתנה הבעייתי (`env -u npm_config_http-proxy`). בסביבה מקומית, אם עדיין מתקבלת האזהרה, מומלץ להריץ פקודות npm עם `env -u npm_config_http-proxy` או לעדכן את משתני הסביבה המקומיים.
2. **מגבלת חיפוש בעברית** — Pagefind מודיע שאין stemming לשפה `he`. החיפוש עובד, אבל התאמות בין נטיות/שורשים בעברית מוגבלות.
3. **ספרים במבנה ללא פרקים בפועל** — בדוח התוכן נראים ספרים עם 0 פרקים (למשל `clementos-geez`, `tsavaat-yeshua`), כנראה שלדי מבנה/placeholder. כדאי להחליט אם להסתיר בניווט עד שיש תוכן.

## הערכת מצב קצרה
מבחינה טכנית הפרויקט במצב טוב להתקדמות: ה-build והבדיקות יציבים, מבנה הקוד ברור, ויש תשתית תוכן וחיפוש פעילה.
הצעדים המיידיים המומלצים: לטפל באזהרת npm, להגדיר אסטרטגיית חיפוש עברית משופרת (או ציפיות UX), ולסגור מדיניות לספרים ללא פרקים.
